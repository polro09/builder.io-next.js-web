# 프로젝트 지침

## 아키텍처 원칙

### 페이지 구조

- **로딩/메인 페이지를 제외한 모든 페이지는 독립적 형태를 지닌다**
  - 각 페이지는 자체적으로 완전한 기능을 수행할 수 있어야 함
  - 페이지 간 의존성을 최소화하여 독립적으로 개발 및 유지보수 가능
  - 로딩 페이지와 메인 페이지만 공통 요소를 공유할 수 있음

## 개발 가이드라인

### 페이지 개발 시 고려사항

1. **독립성**: 각 페이지는 자체 스타일과 로직을 포함
2. **재사용성**: 공통 컴포넌트가 필요한 경우 별도 모듈로 분리
3. **유지보수성**: 페이지별로 명확한 책임 분리

## 프로젝트 구조

```
├── index.html          # 메인 페이지
├── loading.html        # 로딩 페이지 (공통 요소 허용)
├── pages/              # 독립적인 페이지들
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/             # 공통 리소스
    ├── css/
    ├── js/
    └── images/
```

## 개발 서버 실행

```bash
npm run dev
```

서버가 http://localhost:3000 에서 실행됩니다.
